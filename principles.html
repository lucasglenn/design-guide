Design Principles

1. Consistency

Appearance

Across platforms and windows of all sizes, elements must remain as similar as possible as to not interrupt or hinder the user experience. Keeping things like placement, proportions, and elements the same makes interactions seamless.

Behaviour

The interface and its elements should have consistent behaviours across all platforms to facilitate a stress-free and successful experience for the user.

Language

A successful interface is not just consistent internally, but is also consistent with the user's experiences of other products as well.

Consider your user personas' tech fluency during design and development phases. An interface suited to your common-denominator user will result in stress-free use.

The interface must be consistent and coherent to ensure for the following four principles can follow

2. Perceivability

Visibility

Avoid creating hidden interactions, as the user should not have to guess or search for functions. Create a visual hierarchy that compliments the user's needs. The most important interactions should be the most prominent.

Cognitive Load

If there are too many choices, decisions, options the user can get frustrated and be unsure of which one is the "right" choice.

Friction

Breaking information and actions into bite-sized pieces is often better than trying to do the same thing in fewer steps. If each step is too stressful, it can be exhausting and result in a poor overall experience.

If there is an important step that must be understood before continuing, it may be worthwhile to slow the pace down and make sure the user understands before moving on

Hints and Signifiers

Helpful hints and indicators can assist users in interacting seamlessly. This can be as simple as having a higher contrast colour for hyperlinks.

Location and Wayfinding

Don't let the user get lost. Show where they are, where they came from, and they will know where they can go.

3. Predictability

Expectations

If we do our job right, the user's expectations will align with their interactions. For a positive experience, a user should expect what will happen before an interaction occurs. User testing is a great way to refine alignment with user expectations.

Opportunities

Consider all of the opportunities that a user has to interact, each opportunity and subsequent interaction must be perceivable, meaningful, or predictable. Otherwise the result is trial, error, and failure to progress and interact successfully.

4. Feedback-Rich

Feedback

Feedback informs and acknowledges interactions, facilitating a positive, linear workflow. Feedback should not interrupt, but compliment the experience.

Reward or Pay-off

Once a user has invested a significant amount of effort into an experience it is important to reward them with a pay-off or celebration. This can give them a break from the flow and/or provide a chance to feel good about their investment and get ready to keep going.

Error States

When something goes awry, whether it be the fault of the server or the client, feedback is critical. Let the user know what went wrong, why, and how to fix it. Refer to the "Error States" section for more direction

Reduce Latency

Our user's time is important, so don't make them wait. Avoid delays in behaviour and feedback.

5. Learnability

Workflow Pacing

For a quickly learned/repeatable experience, a user should be lead through the experience in a smooth, logical way that feels like there is a manageable goal. Each action should feel like a step closer to completion.

Simplicity and Intuition

The more simple and intuitive your UI, the easier it is to remember your interactions and workflow patterns.

Efficiency

It's all well and good for things to be efficient on the server side, but interactions must be efficient for the user first for productive outcomes. Providing good default settings, keeping things simple, and refining workflow patterns are great ways to do this, and they show respect for your user
